javascript规范

书写javascript规范及jquery一些要注意的地方。以可读性作为首要要求。

主要参考Google JavaScript代码风格指南。

变量声明

变量声明都要加上var关键字

如果你不指定关键字var, 该变量就会暴露在全局作用域（window）中，这很可能会覆盖全局作用域中的同名变量

var speed = data.speed;

参考资料: JavaScript中两种类型的全局对象/函数

参考资料: JavaScript声明全局变量三种方式的异同

全局变量

尽量不使用全局变量。

全局变量使用不当，在合并代码等情况下可能会造成冲突。因此除了存储地图对象的变量，尽量不要使用。如果必须使用时请约定命名空间，规范文档。

var hello = "hello world";//不要这样使用
(function(){
var hello = "hello world";
}())//推荐这样使用

参考资料: 变量对象（Variable object）

命名规范

变量及方法命名均以驼峰式来命名。私有的属性，变量和方法都应该以下划线开头。

变量名应该在让人看懂的基础上尽量短，不要一写一坨。

function getUserSpeed(){
var _initSpeed;
}

注释

方法必须要增加注释。注释的格式按照jsDoc的格式来书写。

注释可不需要按照下面书写的那样复杂，但一定要注意几个必须包含的内容：

@param @argument 指定参数名和说明来描述一个函数参数

@returns 描述函数的返回值

@description 简单描述函数的作用

@author 作者

@version 版本号

/** 
* @fileOverview 简单的类对象标注示例 
* @author llying 
* @version 0.1 
*/ 
/** 
* @author llying 
* @constructor Person 
* @description 一个Person类 
* @see The llying. 
* @example new Parent(“张三”,15); 
* @since version 0.1 
* @param {String} username 姓名 
* @param {Num} age 年龄 
*/ 
function Person(username,age) 
{ 
/** 
* @description {Sting} 姓名 
* @field 
*/ 
this.username = username; 
/** 
* @description {Num} 年龄 
* @field 
*/ 
this.age = age 
/** 
* @description 弹出say内容 
* @param {String} content 内容 
*/ 
this.say = function(content) 
{ 
alert(this.username+" say :"+content); 
} 
/** 
* @description 返回json格式的对象 
* @return {String} json格式 
* @see Person#say 
*/ 
this.getJson = function(){ 
return "{name:"+this.username+",age"+this.age+"}"; 
} 
}

分号

每一语句的结尾都要以分号来结束。

养成良好的习惯，避免代码解析及压缩合并时出现错误。

JS语句以分号作为结束符使得JS解释器解析，如果省略分号，就会由JS解释器确定语句的结尾。

译者注：加上分号也会在某些情况下增进代码的性能，因为这样解析器就不必再花时间推测应该在哪里插入分号了（取自《Javascript高级程序设计》）

var fn=function (){
      alert(123);
    };
     (fn);
var fn=function (){
      alert(123);
    }
     (fn);//两段代码是截然不同的！！

函数声明与函数表达式

要区分函数声明与函数表达式。不要使用块内函数声明。

函数声明与函数表达式的区别请学习参考链接。并且各种引擎对快内函数声明的实现不同。

if (x) {
  function foo() {}
}//只有屌丝才这样做。
if (x) {
  var foo = function() {}
}//非得快内定义请用函数表达式。

基本包装类型

不要使用基本包装类型来创建number string boolean对象。

他的类型判断时会有问题。但可以在类型转换中使用，很好用

var x = Boolean(0);
if (x) {
  alert('hi');  // 不会弹框显示 hi
}
typeof Boolean(0) == 'boolean';//成功转换
typeof new Boolean(0) == 'object';//这里出问题了

参考资料:基本类型的包装对象

内置对象原型

禁止修改内置对象的原型。

随便修改内置对象的原型会遇到很多不可预料的问题。

Array.prototype.a = function(){}//禁止
Object.prototype.b = function(){}//禁止

字面量

推荐使用[],{}的方式代替new Array(),new Object。

可读性会更好。

var a = new Array();//不推荐
var a = [];//推荐
var b = new Object();//不推荐
var b = {};//推荐

参考资料:基本类型的包装对象
